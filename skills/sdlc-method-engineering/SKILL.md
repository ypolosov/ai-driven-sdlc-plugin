# Skill: сквозная ситуативная инженерия методов

## Назначение фазы

Универсальный опрос для выбора уровня SME и инструмента на любой фазе.
Фазовые skills делегируют сюда, не дублируют логику.
Имена инструментов рендерятся runtime из `catalogs/method-tool-matrix.md`.

## Альфы, которые продвигает

- **Way of Working** — уточнение способа исполнения метода.

## Дисциплины

- Situational Method Engineering (`catalogs/disciplines.md`).

## Обязательный интерактивный опрос (принцип 1)

**SME-выбор делает пользователь, не плагин.**

Шаг 0 — ДО записи в `profile.md` или `decisions.md`:
1. Вызвать `AskUserQuestion` с вопросами ниже.
2. Показать 2–3 альтернативы по каждому выбору.
3. Дождаться ответа пользователя; не угадывать уровень и инструмент.
4. Auto mode среды не отменяет интерактивность.
5. Только `/sdlc-autonomy --task hootl` разрешает пропуск опроса.
6. Если `AskUserQuestion` недоступна — остановиться и сообщить пользователю.

## Мета-вопросы (абстрактные, без имён инструментов)

1. Каков масштаб и риск проекта: pet / mid / enterprise?
2. Сколько человек в команде работают над этой фазой?
3. Какая частота релизов/итераций? (раз в день, неделю, месяц)
4. Насколько важна формальная трассируемость и ревью?
5. Какие регуляторные ограничения существуют? (ИБ, комплаенс, GDPR и т. п.)
6. Какой язык/стек выбран пользователем для целевого? (для фазы development)
7. Какие интересы у активной роли? (подтягиваются из `catalogs/roles.md`)

## Варианты ответов и матрица

Варианты уровней SME: `pet` / `mid` / `enterprise`.
Для каждого уровня skill запрашивает у `catalogs/method-tool-matrix.md` строку фазы.
Примеры инструментов представляются пользователю с расчётом на принцип 1:
- Показать 2–3 альтернативы из матрицы.
- Показать возможность добавления своего через `target/.claude/sdlc/method-tool-extensions.md`.
- Учесть интересы активной роли (Левенчук Том 1 гл. 6).

## Протокол инстанцирования

1. Skill фазы передаёт контекст: `phase`, активная роль, текущие альфы.
2. **Обязательно:** в HITL/HOTL вызвать `AskUserQuestion`; в HOOTL — эвристика + запись альтернатив в `decisions.md`.
3. По ответам определить уровень SME (pet/mid/enterprise).
4. Прочитать строку матрицы по фазе и уровню.
5. Показать варианты инструментов (из плагина + extensions целевого).
6. Зафиксировать выбор в `profile.md` (строка фазы) и в `decisions.md` (с альтернативами).
7. Вернуть выбор вызвавшему skill для инстанцирования шаблона.

## Критерии готовности

- `profile.md` обновлён: строка фазы содержит уровень, метод, инструмент.
- `decisions.md` содержит запись с ≥2 альтернативами.
- Выбранный инструмент существует в `method-tool-matrix.md` или в `method-tool-extensions.md`.

## Следующие шаги по ролям

- Любая роль → продолжить фазу через `/sdlc-phase <name>` — skill фазы получит выбор.
- method-engineer → можно перезапустить выбор командой `/sdlc-phase <name> --reconfigure`.
