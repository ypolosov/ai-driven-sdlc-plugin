# Skill: перенос внимания на систему

## Назначение фазы

Перенос внимания на выбранный системный уровень.
Помогает определить относительно новой целевой: надсистему, подсистемы, окружение, систему создания.
Сохраняет фокус в `system-context.md`.

## Альфы, которые продвигает

- **Software System** — уточнение границ целевой системы.

## Дисциплины

- Systems thinking (`catalogs/disciplines.md`).

## Мета-вопросы

1. На какую систему вы хотите перенести внимание? (slug или свободное описание)
2. Эта система представлена директорией в репозитории (`materialized`) или абстракцией (`logical`)?
3. Какова её роль относительно прежней целевой? (надсистема / подсистема / окружение / система создания / новая целевая)
4. Это постоянный перенос или разовый (флаг `--transient`)?
5. Нужно ли выйти из рассмотрения какой-то прежней системы (флаг `--retire <slug>`)?

Конкретные инструменты моделирования не выбираются здесь; см. фазу architecture.

## Матрица SME

Применяется `catalogs/method-tool-matrix.md#architecture` при необходимости визуализации.
На этапе фокуса моделирование опционально.

## Протокол инстанцирования

1. Прочитать `system-context.md` целевого.
2. Применить действие (focus / transient / retire) к записи.
3. Пересчитать `role_vs_target` для связанных систем.
4. Обновить `current_focus` и журнал фокусировок.
5. В Волне 2: создать/актуализировать описание (`README.sdlc.md` или `external-systems/<slug>.md`).
6. Зафиксировать изменение в `decisions.md` (с 2–3 альтернативами, если переход неочевиден).

## Критерии готовности

- `system-context.md` согласован: одна система с `role_vs_target: target`.
- `current_focus` в frontmatter указывает на неё.
- Для `materialized`-систем в Волне 2 — существует `README.sdlc.md`.
- `check-cross-refs.sh` не находит осиротевших ссылок на retired-системы.

## Следующие шаги по ролям

- systems-thinker → `/sdlc-phase architecture` для моделирования новой целевой.
- product-owner → `/sdlc-phase vision` если смена целевой меняет ценность.
- architect → проверка согласованности с архитектурными артефактами через `/sdlc-audit`.
