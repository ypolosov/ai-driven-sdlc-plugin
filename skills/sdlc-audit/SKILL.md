# Skill: сквозной аудит консистентности

## Назначение фазы

Проверка согласованности артефактов SDLC между фазами и с профилем.
Детерминированная часть — в скриптах; семантика — в агенте `sdlc-consistency-auditor`.
Результат — `audit.md` в целевом.

## Альфы, которые продвигает

- Все альфы — аудитор сверяет их состояние с артефактами через `sdlc-alpha-tracker`.

## Дисциплины

- Systems thinking (`catalogs/disciplines.md`).

## Мета-вопросы

При ручном запуске через `/sdlc-audit`:
1. Запустить полный аудит или только по конкретной фазе/альфе?
2. Режим `--apply` — предлагать фиксы интерактивно (с 2–3 альтернативами)?
3. Критичность: показать только `fail` или все расхождения?

## Проверки

### Детерминированные (скрипты)

- `check-cross-refs.sh` — осиротевшие ссылки в `.claude/sdlc/**`.
- `validate-artifact.sh` — frontmatter, секции, ≤15 слов (с исключениями принципа 4).

### LLM-шаги (агент)

- **Трассируемость между фазами.** Артефакт фазы N+1 ссылается на фазу N.
- **Соответствие уровню SME.** Артефакты соответствуют мета-шаблону уровня из `profile.md`.
- **Альфы ↔ артефакты.** Продвижение альфы подтверждено артефактом (через `sdlc-alpha-tracker`).
- **System-context ↔ артефакты архитектуры.** Сущности в `system-context.md` согласованы.
- **TDD-семантика.** Тест описывает поведение из diff (второй слой принципа 5).

## Протокол инстанцирования отчёта

1. Запустить детерминированные скрипты, собрать их выводы.
2. Запустить `sdlc-consistency-auditor` с входами: `CLAUDE.md` плагина, `profile.md`, `alphas.md` через трекер, `system-context.md`, артефакты `.claude/sdlc/**`.
3. Сформировать `audit.md` по `audit-report.meta.md` — со статусом и списком расхождений.
4. Для каждого расхождения сформулировать 2–3 альтернативы фикса (принцип 1).
5. При `--apply` — интерактивно применить выбранный фикс; фиксация в `decisions.md`.

## Критерии готовности

- `audit.md` перезаписан в `.claude/sdlc/`.
- Статус `pass` — все проверки прошли.
- Статус `warn`/`fail` — перечислены расхождения с фиксами.

## Следующие шаги по ролям

- Любая роль → разобрать расхождения через `/sdlc-audit --apply`.
- method-engineer → если расхождений много, возможно, уровень SME выбран неверно.
- architect → проверить System-context и архитектуру.
