# `/sdlc-continue`

Ориентирует пользователя в текущем состоянии проекта.
Подсказывает, какую фазу запустить исходя из роли и состояния альф.

## Поведение

1. Считать активную роль из `roles.md` или спросить пользователя (выбор из `catalogs/roles.md`).
2. Вызвать `sdlc-state-reader` — получить текущее состояние альф и фаз.
3. Определить подходящие фазы по схеме `role → [phases] → [alphas обязательные]`.
4. Предложить 2–3 альтернативы (принцип 1).
5. Зафиксировать выбор в `decisions.md` и запустить соответствующий `/sdlc-phase` или skill.

## Аргументы

- `--role <slug>` — предустановить роль без вопроса (например, `--role developer`).

## Пример взаимодействия

- Роль: developer.
- Состояние: Requirements=Bounded, Software System=Architecture Selected, нет тестов.
- Предложение:
  1. `/sdlc-phase testing` (TDD-first, рекомендация).
  2. `/sdlc-phase development` (если тесты уже есть в плане).
  3. `/sdlc-focus` на подсистему для уточнения границ.

## Связи

- Использует `sdlc-state-reader` для чтения состояния.
- Использует `sdlc-alpha-tracker` для получения состояния альф.
- Использует `catalogs/roles.md` для привязки роль → фазы → альфы.
