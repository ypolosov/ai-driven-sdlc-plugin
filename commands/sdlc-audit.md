# `/sdlc-audit`

Активирует skill `sdlc-audit` для сквозной проверки консистентности.

## Поведение

1. Запустить детерминированные скрипты:
   - `check-cross-refs.sh` — осиротевшие ссылки.
   - `validate-artifact.sh` — frontmatter и структура артефактов.
2. Запустить LLM-агент `sdlc-consistency-auditor` со всеми входами.
3. Сформировать отчёт по `audit-report.meta.md` → `<target>/.claude/sdlc/audit.md`.
4. При `--apply` — для каждого расхождения показать 2–3 альтернативы фикса (принцип 1) и применить выбранную.

## Аргументы

- `--scope <phase>` — ограничить проверку одной фазой.
- `--apply` — предложить фиксы интерактивно (в HOOTL фикс выбирается автономно, пишется в `decisions.md`).

## Проверяет

- Трассируемость артефактов между фазами.
- Соответствие артефактов уровню SME из `profile.md`.
- Продвижение альф подтверждено артефактами (через `sdlc-alpha-tracker`).
- Согласованность `system-context.md` с артефактами архитектуры.
- Осиротевшие ссылки.
- TDD-семантика (второй слой принципа 5).

## Статусы

- `pass` — всё согласовано.
- `warn` — есть заметки, не блокирует.
- `fail` — есть блокирующие расхождения (рекомендуется не коммитить до фикса).

## Связи

- Использует `sdlc-consistency-auditor` и `sdlc-alpha-tracker`.
- Запускается также PostToolUse async после записи в `.claude/sdlc/**`.
