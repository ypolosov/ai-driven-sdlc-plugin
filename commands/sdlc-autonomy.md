# `/sdlc-autonomy`

Активирует skill `sdlc-autonomy` для установки уровня взаимодействия.

## Поведение

1. Разобрать уровень: `hitl` / `hotl` / `hootl`.
2. Если `--phase <name>` — обновить `default_autonomy` для фазы в `profile.md`.
3. Если `--task` — установить ephemeral override на текущую задачу.
4. Без флагов — выставить уровень для всех фаз в `profile.md`.
5. Зафиксировать в `decisions.md` (принцип 1).

## Аргументы

- `<level>` — уровень автономности:
  - `hitl` — подтверждение на каждый шаг.
  - `hotl` — наблюдение с возможностью остановки.
  - `hootl` — полная автономия с логом альтернатив.
- `--phase <name>` — ограничить только одной фазой.
- `--task` — применить только к текущей задаче, не сохранять в файл.

## Важно

- HOOTL без предварительного согласия на автоматизацию опасных действий требует подтверждения.
- Принцип 1 работает всегда: альтернативы порождаются и пишутся в `decisions.md`.
- В HOOTL альтернативы не предъявляются, но логируются.

## Пример

```
/sdlc-autonomy hotl --phase development
/sdlc-autonomy hootl --task
```

## Связи

- Использует skill `sdlc-autonomy`.
- Hooks считывают уровень из `profile.md` и переменной сессии.
