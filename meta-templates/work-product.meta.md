# Мета-шаблон рабочего продукта (базовый)

Все рабочие продукты в `<target>/.claude/sdlc/` наследуют эту схему.

## Обязательный frontmatter

```yaml
---
name: <slug артефакта>
type: <vision|requirements|architecture|testing|deployment|operations|cross-cutting>
phase: <фаза SDLC, если применимо>
sme_level: <pet|mid|enterprise>
method: <выбранный метод из catalogs/method-tool-matrix.md>
tool: <выбранный инструмент>
alphas: [ссылки на альфы]
role: <создавшая роль>
created: <YYYY-MM-DD>
updated: <YYYY-MM-DD>
system_of_attention: <slug из system-context.md>
---
```

## Обязательные секции

1. **Назначение** — одно утверждение, зачем существует артефакт.
2. **Привязка к методу** — ссылка на строку матрицы и `method_abstract`.
3. **Содержание** — ядро артефакта по шаблону инструмента.
4. **Связи с другими артефактами** — ссылки на артефакты других фаз.
5. **Критерии готовности** — fitness-функции, по которым считается завершённым.

## Правила записи

- Язык — русский.
- Каждое утверждение содержит не более 15 слов.
- Цитаты из первоисточников в fenced-блоках не учитываются.
- Код без комментариев.

## Валидация

Скрипт `validate-artifact.sh` проверяет:
- Наличие всех обязательных полей frontmatter.
- Наличие всех 5 обязательных секций.
- Соответствие правилу ≤15 слов.
- Русский язык вне блоков цитат.
