# Мета-шаблон артефакта фазы SDLC

Используется skills фаз при инстанцировании конкретного шаблона в целевом проекте.

## Операциональный тест абстрактности

Шаблон считается абстрактным, если выполнены три условия:
- a) Имена разделов не содержат названий инструментов или фреймворков.
- b) Каждая секция описывается через альфу, роль или дисциплину из `catalogs/`.
- c) Мета-шаблон инстанцируется под ≥2 разных инструмента без изменений.

## Frontmatter (расширяет work-product.meta)

```yaml
---
name: <slug>
type: <фаза SDLC>
phase: <vision|requirements|architecture|development|testing|deployment|operations>
sme_level: <pet|mid|enterprise>
method: <строка из catalogs/method-tool-matrix.md>
tool: <выбранный пользователем инструмент>
alphas: [продвигаемые альфы]
disciplines: [задействованные дисциплины]
role: <создавшая роль>
traces_from: [артефакты предыдущей фазы]
traces_to: [артефакты последующей фазы]
system_of_attention: <slug>
created: <YYYY-MM-DD>
updated: <YYYY-MM-DD>
---
```

## Обязательные секции

### 1. Назначение
Одно утверждение, какой альфы продвижение и какой метод используется.

### 2. Привязка к фазе и методу
Ссылки на фазу, уровень SME, дисциплину, выбранный инструмент.

### 3. Содержание
Ядро артефакта, структура подбирается по выбранному инструменту через `context7`.
Содержание всегда привязано к альфам и ролям, не к инструменту напрямую.

### 4. Трассируемость
Ссылки `traces_from` и `traces_to` на артефакты соседних фаз.
Продвигаемая альфа ссылается на текущее состояние в `alphas.md`.

### 5. Критерии готовности
Fitness-функции артефакта (скрипты или LLM-проверки).
Пример на abstract-уровне: «все критерии приёмки выполнены», «сценарий пройден».

## Правила инстанцирования

- Skill фазы читает выбор пользователя из опроса `sdlc-method-engineering`.
- Skill фазы вызывает `context7` для референсной структуры выбранного инструмента.
- Skill фазы объединяет референс с этим мета-шаблоном, сохраняя обязательные секции.
- Конкретный шаблон пишется в `<target>/.claude/sdlc/phases/<phase>/<artifact>.md`.

## Запрещено в плагине

- Конкретные структуры Lean Canvas, arc42, ADR, User Story, TODO.md.
- Имена инструментов вне `catalogs/method-tool-matrix.md`.
