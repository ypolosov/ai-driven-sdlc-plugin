# Мета-шаблон credentials

Секреты целевого проекта хранятся в `.env` в корне.
`.env` обязательно добавлен в `.gitignore`.
Плагин поставляет `.env.example` и правило `.gitignore`.

## Файл `.env.example`

Пример без значений; только имена переменных с короткими комментариями-строками через `#`.

```
# Пример .env.example — значения не коммитить в git
# Скопировать в .env и заполнить

# GitHub API (для state-артефакта через GitHub Issues, опционально)
# GITHUB_TOKEN=

# Прочие секреты целевой системы добавляются по ходу фаз
```

Комментарии в `.env.example` допустимы (принцип 4a распространяется на исходный код).

## Дополнение к `.gitignore`

Если в `.gitignore` нет строки `.env` — плагин добавляет её.
Блок добавляется с заголовком-маркером, чтобы плагин мог найти его при переустановке.

```
# ai-driven-sdlc: credentials (принцип 10) — не коммитить секреты
.env
.env.local
.env.*.local
```

## Правила

- `.env.example` создаётся только если его нет.
- `.env` никогда не создаётся плагином.
- При обнаружении `.env` в индексе git плагин останавливается и предупреждает.
- Значения секретов плагин не запрашивает и не читает.

## Валидация

- `bootstrap-target.sh` проверяет наличие `.env` в `.gitignore`.
- `/sdlc-audit` проверяет: `.env.example` существует и не содержит значений вида `=<непустое>`.
