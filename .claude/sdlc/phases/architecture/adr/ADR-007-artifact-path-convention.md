# ADR-007: Конвенция пути артефактов SDLC

## Контекст

Hooks и агенты должны однозначно опознавать артефакты плагина.
Смешение с другими `.claude/` файлами ведёт к ложным срабатываниям.
Нужно чёткое разделение SDLC и прочей конфигурации Claude Code.

## Решение

Все артефакты SDLC живут в `<target>/.claude/sdlc/`.
Hooks и агенты опираются только на этот префикс.
Имя state-артефакта может быть вне `.claude/`; фиксируется в `plugin-config.md`.
`/sdlc-init` поддерживает режимы: fail-if-exists, merge, force.

## Последствия

- Однозначная граница SDLC-артефактов в репозитории.
- Hook-фильтры просты и надёжны.
- Корневой README и прочие артефакты вне SDLC не трогаются.
- Переход на новый плагин сводится к копированию одной директории.

## Альтернативы

1. Корень репозитория — отвергнуто, конфликты имён.
2. Отдельный каталог вне `.claude/` — отвергнуто, нарушает Claude Code конвенции.
