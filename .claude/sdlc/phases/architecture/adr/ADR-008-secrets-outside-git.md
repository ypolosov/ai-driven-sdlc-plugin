# ADR-008: Секреты целевого проекта вне git

## Контекст

AI-агенты генерируют artefacts с риском случайной фиксации секретов.
Токены MCP github, API-ключи, SMTP-пароли не должны попадать в историю.
Обнаружение утечки в public-репозитории требует ротации токенов.

## Решение

Секреты хранятся в `<target>/.env` в корне целевого проекта.
Плагин поставляет `.env.example` без значений и правило `.gitignore`.
`/sdlc-init` добавляет строки `.env`, `.env.local`, `.env.*.local` в `.gitignore`.
Артефакты SDLC не содержат токенов; ссылки идут через `${VAR_NAME}`.

## Последствия

- Скомпрометированные токены ротируются локально без истории.
- MCP-серверы получают credentials через env-var подстановку.
- Pre-commit hooks могут сканировать diff на формат токенов.
- Пользователь отвечает за не-ручное добавление секретов.

## Альтернативы

1. Секреты в конфиге плагина — отвергнуто, попадают в git.
2. Внешний vault как обязательная зависимость — отвергнуто, чрезмерно для pet.
