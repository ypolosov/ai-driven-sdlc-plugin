# ADR-001: Методологический каркас, не технология

## Контекст

AI-инструменты SDLC навязывают конкретные стеки и фреймворки.
Пользователи Claude Code работают с разными языками и процессами.
Плагин должен применяться к любой IT-системе без навязывания технологий.

## Решение

Плагин несёт только каркас: альфы, дисциплины, методы, роли, мета-шаблоны, матрицу.
Конкретные шаблоны инструментов инстанцируются в целевом проекте.
Запрещены внутри плагина: Lean Canvas, arc42, ADR-шаблоны, User Story, TODO.md.
Обоснование: Левенчук Том 1 гл. 4 «Воплощение и описание», Том 2 гл. 10.

## Последствия

- Пользователь свободен в выборе инструментов через matrix + extensions.
- Референсная структура инструмента подтягивается через context7.
- Плагин переносим между проектами без адаптации.
- Нет одной стоп-кнопки «запустить стандартный workflow».

## Альтернативы

1. Жёсткий Scrum/SAFe workflow встроен в плагин — отвергнуто, нарушает принцип 3.
2. Богатый набор готовых шаблонов — отвергнуто, плодит дубли и легаси.
