# ADR-003: Ситуативная инженерия методов по фазам

## Контекст

Жёсткий уровень сложности на весь проект плохо масштабируется.
Один проект может иметь pet-vision и mid-testing.
Пользователь должен выбирать уровень сложности под контекст.

## Решение

Уровень сложности pet/mid/enterprise выбирается по каждой фазе независимо.
Выбор фиксируется в `profile.md` целевого проекта.
SME-опрос ведёт агент `sdlc-method-engineer` через абстрактные мета-вопросы.
Пользователь добавляет свои инструменты в `method-tool-extensions.md`.

## Последствия

- Метод адаптируется к фазе, не навязывает единый уровень.
- SME-опрос — одинаковый код, разные варианты из матрицы.
- Скачки между уровнями фиксируются в `decisions.md` и истории profile.
- Необходим runtime-рендер вариантов из matrix + extensions.

## Альтернативы

1. Единый уровень на весь проект — отвергнуто, слишком грубо.
2. Выбор только из жёсткого списка — отвергнуто, нет расширяемости.
