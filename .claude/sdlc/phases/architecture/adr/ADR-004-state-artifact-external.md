# ADR-004: State-артефакт вне плагина

## Контекст

Плагин не должен держать состояние целевого проекта.
Формат трекинга задач варьируется: файл, каталог, GitHub Issues.
Пользователь должен выбирать формат под команду и инструменты.

## Решение

Состояние работ живёт в state-артефакте, указанном пользователем.
Имя и тип артефакта фиксируются в `plugin-config.md`.
Поддерживаются kinds: `file`, `dir`, `mcp`.
Без `state_artifact` агент `sdlc-state-reader` падает с сообщением.

## Последствия

- Плагин переносим без миграции состояния.
- Возможна интеграция с GitHub Issues через MCP.
- Пользователь отвечает за согласованность state-артефакта.
- Смена формата — правка одной строки конфига.

## Альтернативы

1. Внутреннее состояние в JSON плагина — отвергнуто, плодит sync-задачи.
2. Только markdown-файл — отвергнуто, не масштабируется.
