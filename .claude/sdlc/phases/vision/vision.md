# Vision плагина ai-driven-sdlc

## 1. Назначение

Продвинуть альфу Opportunity в состояние Value Established.
Продвинуть альфу Stakeholders в состояние Represented.
Метод — одностраничное описание проблемы и ценности.

## 2. Привязка к фазе и методу

- Фаза: vision.
- Уровень SME: pet.
- Дисциплина: product-discovery; stakeholder-analysis.
- Инструмент: README-as-vision (корневой README несёт цель и проблему).

## 3. Содержание

### 3.1. Проблема

AI-агенты генерируют код без осмысления системы, надсистемы и стейкхолдеров.
Решения между альтернативами теряются в чатах и не фиксируются.
Жёсткие SDLC-фреймворки навязывают церемонии вне зависимости от контекста.
Без метода качество AI-разработки не воспроизводится от проекта к проекту.

### 3.2. Целевой бенефициар

AI-first соло-разработчики, применяющие Claude Code для SDLC-осмысленной разработки.
Вторичные — малые команды с AI-агентами; инженеры методов.

### 3.3. Предлагаемое решение

Методологический каркас SDLC как Claude Code плагин.
Каркас не навязывает инструменты; помогает выбирать их ситуативно.
Каркас фиксирует альтернативы и выбор как артефакты Claude Code.
Каркас применим к любой IT-системе, включая себя (dogfooding).

### 3.4. Ключевые стейкхолдеры и интересы

- AI-first соло-разработчик — скорость с сохранением метода.
- Инженер методов — соответствие метода ситуации, эволюция практик.
- Системный мыслитель — корректные границы системы и надсистемы.
- Автор плагина (dogfooding) — доказательство ценности применением к себе.

Полные определения ролей — в `catalogs/roles.md` плагина.

### 3.5. Альтернативы в нише

- Ручные CLAUDE.md и slash-commands — без методологического каркаса.
- Классические SDLC-фреймворки (Scrum, SAFe, arc42) — без AI-интеграции.
- Узкие AI-плагины (Cursor rules, Aider) — без покрытия всего SDLC.
- Системное мышление Левенчука и Essence — теория без инструментальной поддержки.

### 3.6. Отличие плагина

Ситуативная инженерия методов по фазам, а не жёсткий процесс.
Фиксация альтернатив как first-class артефактов Claude Code.
Dogfooding — плагин применим к себе как целевой системе.
Основа — системное мышление Левенчука и OMG Essence.

### 3.7. Противоречия интересов

- Скорость против методичности — решается автономией HOTL/HOOTL по задаче.
- Автономия против контроля — default HITL, override на фазу или задачу.
- Простота против полноты — уровни SME pet/mid/enterprise по фазам независимо.
